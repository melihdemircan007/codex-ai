# PR Readiness Checklist

- [ ] Jira acceptance criteria are mapped to implementation and tests.
- [ ] Grill-Me decisions are captured in the Jira Development Log.
- [ ] Scope is limited to the requested behavior.
- [ ] Tests were added or updated for changed behavior.
- [ ] Targeted tests passed locally.
- [ ] Affected module build passed locally or the blocker is documented.
- [ ] Self-review found no unresolved critical or high issues.
- [ ] Jira Development Log includes test evidence and remaining risks.
- [ ] Bitbucket PR description includes Jira key, summary, tests, risks, and QA notes.
- [ ] No secrets or internal tokens were committed or pasted into repo files.
